通过Hugo生成个人主页，然后托管到GitHub Page

托管个人/组织的页面

你必须使用 username.github.io 这样的命名形式。 master 分支的内容将会用于构建和发布到你的 GitHub Pages 站点。 这种情况下就简单多了：我们将创建两个不同的仓库，一个用于托管 Hugo 的内容，其中有一个 git 子模块的 public 文件夹。

一步一步来：

  1. 在 GitHub 上创建 -hugo 代码仓库（它将用于托管 Hugo 的内容）
  2. 在 GitHub 上创建 .github.io 代码仓库（它将用于托管 public 文件夹：静态的网站）
  3. git clone <<your-project>-hugo-url> && cd <your-project>-hugo
  4. 让你的网站在本地生效（hugo server --watch -t <yourtheme>）
  5. 当你对结果满意时，Ctrl+C (关闭服务)并 rm -rf public（不用着急，使用 hugo -t <yourtheme> 总会重新生成这个文件夹的）
  6. git submodule add git@github.com:<username>/<username>.github.io.git public
  7. 添加 deploy.sh 脚本来帮你（让它可执行：chmod +x deploy.sh）
  8. 运行./deploy.sh "Your optional commit message"把变动推送到 .github.io（小心，你也许想要提交变化到 -hugo 仓库）。

就是这样！你的个人页面运行在 http://username.github.io/（在10分钟延迟之后）。

#!/bin/bash

echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"

# Build the project.
hugo # if using a theme, replace by `hugo -t <yourtheme>`

# Go To Public folder
cd public
# Add changes to git.
git add -A

# Commit changes.
msg="rebuilding site `date`"
if [ $# -eq 1 ]
  then msg="$1"
fi
git commit -m "$msg"

# Push source and build repos.
git push origin master

# Come Back
cd ..
